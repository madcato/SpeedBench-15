## Part 1: Transformer Architecture

The transformer architecture, introduced by Vaswani et al. in the 2017 paper "Attention Is All You Need," revolutionized natural language processing and became the foundation for virtually all modern large language models. Unlike its predecessors — recurrent neural networks (RNNs) and long short-term memory networks (LSTMs) — the transformer eschews sequential processing in favor of parallelism, making it far more efficient on modern hardware accelerators such as GPUs and TPUs.

At the heart of the transformer is the self-attention mechanism. Given a sequence of token embeddings, self-attention computes, for each token, a weighted sum of all other tokens in the sequence. The weights are derived from the dot product of query (Q) and key (K) vectors, scaled by the square root of the key dimension to prevent vanishing gradients, and then passed through a softmax function. These attention weights are applied to value (V) vectors. Formally:

Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) * V

Multi-head attention extends this by running several attention heads in parallel, each learning different relational patterns. The outputs are concatenated and projected through a learned linear layer. This allows the model to attend simultaneously to different aspects of the input — syntactic structure, semantic meaning, coreference, and so on.

A transformer block consists of a multi-head self-attention sublayer followed by a position-wise feed-forward network (FFN), with residual connections and layer normalization applied around each sublayer. The FFN typically expands the hidden dimension by a factor of four before projecting back down, introducing nonlinearity through an activation function such as ReLU or GeLU.

Since transformers process all tokens in parallel, they require a mechanism to encode positional information. The original paper used fixed sinusoidal positional encodings; modern models increasingly use rotary positional embeddings (RoPE) or ALiBi (Attention with Linear Biases) to support longer context windows more gracefully.

## Part 2: Pre-training Objectives

Large language models are typically trained using one of two primary objectives. Autoregressive (or causal) language modeling, used by GPT-family models, trains the model to predict the next token in a sequence given all preceding tokens. The model never sees future tokens during training, enforced by a causal attention mask. This approach naturally yields models well-suited for open-ended text generation.

Masked language modeling (MLM), introduced by BERT, randomly masks a fraction of input tokens (typically 15%) and trains the model to predict the masked tokens from their surrounding context. This bidirectional objective produces richer contextual representations but does not directly train generation capabilities.

Modern instruction-tuned models such as GPT-4, Claude, and Gemini begin with causal pre-training on massive corpora of text, then undergo supervised fine-tuning (SFT) on curated question-answer pairs, followed by reinforcement learning from human feedback (RLHF) or related methods such as direct preference optimization (DPO). RLHF trains a reward model on human comparisons of model outputs, then uses that reward signal to fine-tune the policy model via proximal policy optimization (PPO) or similar algorithms.

## Part 3: Scaling Laws

Research by Kaplan et al. (2020) established empirical scaling laws for language model performance: model loss decreases as a power law with respect to model size (number of parameters), dataset size, and compute budget. This motivated the construction of increasingly large models, culminating in models with hundreds of billions of parameters.

Chinchilla scaling laws (Hoffmann et al., 2022) refined this understanding, demonstrating that for a given compute budget, prior large models were significantly undertrained. The optimal strategy is to scale model size and training data proportionally. For instance, a 70-billion parameter model requires roughly 1.4 trillion tokens of training data to be compute-optimal, compared to earlier assumptions of ~300 billion tokens.

## Part 4: Tokenization

Modern LLMs use sub-word tokenization, most commonly byte-pair encoding (BPE) or SentencePiece variants. BPE begins with individual characters as the vocabulary and iteratively merges the most frequent adjacent pair until a desired vocabulary size is reached (commonly 32,000–128,000 tokens). This balances coverage of rare words (represented as multiple tokens) and efficiency for common words (represented as single tokens).

Tokenization has significant implications for model performance. Tasks involving character-level operations (counting letters, reversing strings) are inherently harder because the model must reason across token boundaries rather than individual characters. Some languages with complex morphology or non-Latin scripts are tokenized less efficiently, requiring more tokens per word.

## Part 5: Context Windows and KV Cache

The context window defines the maximum number of tokens a transformer can attend to in a single forward pass. Early models had context windows of 512 or 1024 tokens. Modern models support 128K tokens or more, enabled by architectural improvements (efficient attention variants like Flash Attention) and positional embedding techniques that extrapolate to longer sequences.

The KV cache is a critical optimization for inference. During autoregressive generation, each new token must attend to all previous tokens. Without caching, this requires recomputing the key and value matrices for every past token at each step — O(n^2) in sequence length. The KV cache stores these matrices after each forward pass and reuses them, reducing per-step computation to O(n) for the new token only. This dramatically reduces latency and enables practical deployment of long-context models.

KV cache size scales with sequence length, number of layers, number of attention heads, and head dimension. A long conversation with a large model can require gigabytes of cache memory, motivating research into cache compression techniques such as grouped query attention (GQA), which reduces the number of key-value heads (e.g., from 64 to 8) while keeping query heads unchanged, substantially reducing cache memory without significant quality loss.

## Part 6: Inference Optimization

Several techniques improve inference throughput:

**Quantization** reduces the numerical precision of weights from 32-bit or 16-bit floats to 8-bit integers (INT8) or even 4-bit (INT4), reducing memory footprint and often improving throughput on hardware with efficient integer arithmetic units. Post-training quantization (PTQ) applies quantization after training; quantization-aware training (QAT) incorporates it during training for better quality.

**Speculative decoding** uses a small draft model to generate multiple tokens quickly, then verifies them with the full model in a single parallel forward pass. Accepted tokens are kept; rejected tokens are discarded and regenerated. Throughput improvements of 2–4x are achievable when the draft model predicts well.

**Continuous batching** (also called iteration-level scheduling) allows inference servers to add new requests to a batch mid-generation, rather than waiting for the entire batch to complete. This significantly improves GPU utilization and throughput in production settings.

**Flash Attention** (Dao et al., 2022) is an exact attention algorithm that reorders computation to maximize use of fast on-chip SRAM, reducing memory bandwidth usage from O(n^2) to O(n) and enabling attention over very long sequences on GPUs.
